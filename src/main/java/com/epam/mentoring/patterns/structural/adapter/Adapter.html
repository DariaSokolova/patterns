Adapter

Description

Converts the interface of a class into another interface clients expect. Lets classes work together that couldn't otherwise because of incompatible interfaces.
Should be used when there is a need to use an existing class, and its interface does not match the one needed; to create a reusable class that cooperates with unrelated or unforeseen classes; there is a need to use several existing subclasses, but it's impractical to adapt their interface by subclassing every one. An object adapter can adapt the interface of its parent class.

Using in java API:

java.io.OutputStreamWriter(OutputStream out)
java.util.Collections#list()

Pros:

  • Helps achieve reusability and flexibility.
  • Can interconnects two incompatible interfaces.
  • Allows to use new APIs (with different interfaces) and to not break existing implementations.

Cons:

  • Aren't transparent to all clients (can be fixed by using two-way adapters).
  • All requests are forwarded.
  • Some number of adaptations may be needed on an adapter chain to reach the type which is required.
