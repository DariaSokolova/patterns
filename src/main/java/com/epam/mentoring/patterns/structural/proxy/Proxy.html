Proxy

Description

Provides a surrogate or placeholder for another object to control access to it.
Should be used when there is a need to have a a local representative for an object in a different address space; to create expensive objects on demand; to control access to the original object; to have a replacement for a bare reference that performs additional actions when an object is accessed.

Using in java API:

java.lang.SecurityManager
java.io.InputStreamReader

Pros:

  • Can perform optimizations such as creating an object on demand.
  • Can provide security checks.
  • Allows additional tasks when an object is accessed.

Cons:

  • Can hide the fact that an object resides in a different address space.
  • Less efficiency due to indirection.
