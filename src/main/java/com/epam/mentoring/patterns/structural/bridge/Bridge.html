Bridge

Description

Decouples an abstraction from its implementation so that the two can vary independently.
Should be used where there is a need to avoid a permanent binding between an abstraction and its implementation; both the abstractions and their implementations should be extensible by subclassing; changes in the implementation of an abstraction should have no impact on clients; to share an implementation among multiple objects, and this fact should be hidden from the client.

Using in java API:

JDBC API
AWT API

Pros:

  • Decoupling interface and implementation.
  • Improves extensibility.
  • Hiding implementation details from clients.

Cons:

  • Increases complexity.
  • Double indirection - impact on performance.
