Decorator

Description

Attaches additional responsibilities to an object dynamically. Decorators provide a flexible alternative to subclassing for extending functionality.
Should be used to add responsibilities to individual objects dynamically and transparently, that is, without affecting other objects; for responsibilities that can be withdrawn; when extension by subclassing is impractical.

Using in java API:

javax.servlet.http.HttpServletRequestWrapper
javax.servlet.http.HttpServletResponseWrapper

Pros:

  • Has more flexibility than inheritance.
  • Avoids feature-laden classes high up in the hierarchy.

Cons:

  • A decorator and its component aren't identical.
  • Lots of little objects.
