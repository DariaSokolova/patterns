Flyweight

Description

Uses sharing to support large numbers of fine-grained objects efficiently.
Should be used when an application uses a large number of objects; storage costs are high because of the sheer quantity of objects; most object state can be made extrinsic; many groups of objects may be replaced by relatively few shared objects once extrinsic state is removed; the application doesn't depend on object identity.

Using in java API:

java.lang.Integer#valueOf(int)
java.lang.String.intern()

Pros:

  • Reduces the amount of memory used.
  • Reduces the number of objects.

Cons:

  • Time costs on object transferring and finding.
