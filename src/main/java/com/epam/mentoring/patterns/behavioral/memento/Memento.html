Memento

Description

Without violating encapsulation, capture and externalize an object'sinternal state so that the object can be restored to this state later.
Should be used when a snapshot of (some portion of) an object's state must be saved so that it can be restored to that state late; a direct interface to obtaining the state would expose implementation details and break the object's encapsulation.

Using in java API:

java.io.Serializable (implementations)
javax.faces.component.StateHolder (implementations)

Pros:

  • Preserving encapsulation boundaries.
  • Simplifies Originator.

Cons:

  • Using mementos might be expensive.
  • Hidden costs in caring for mementos.
