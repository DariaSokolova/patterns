Visitor

Description

Represents an operation to be performed on the elements of an object structure. Visitor lets you define a new operation without changing the classes of the elements on which it operates.
Should be used when an object structure contains many classes of objects with differing interfaces, and there is a need to perform operations on these objects that depend on their concrete classes; many distinct and unrelated operations need to be performed on objects in an object structure, and you want to avoid "polluting" their classes with these operations; the classes defining the object structure rarely change, but you often want to define new operations over the structure.

Using in java API:

java.lang.reflect.Array#newInstance()
java.util.Calendar#getInstance()

Pros:

  • Makes adding new operations easy.
  • Gathers related operations and separates unrelated ones.
  • Accumulating state.

Cons:

  • Adding new ConcreteElement classes is hard.
  • Difficulties in visiting across class hierarchies.
  • Breaking encapsulation.
