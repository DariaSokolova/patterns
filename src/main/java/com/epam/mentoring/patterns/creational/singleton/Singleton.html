Singleton

Description

Ensures that a class only has one instance, and provide a global point of access to it.
Should be used when there must be exactly one instance of a class, and it must be accessible to clients from a well-known access point; when the sole instance should be extensible by subclassing, and clients should be able to use an extended instance without modifying their code.

Using in java API:

java.lang.Runtime#getRuntime()
java.lang.System#console()

Pros:

  • Controlled access to sole instance.
  • Reduced name space as it is an improvement over global variables.
  • Permits a variable number of instances.

Cons:

  • Generally violates the single responsibility principle: they control their own creation and lifecycle (but it can be delegated to another class).
  • Limit the flexibility of the program: the singleton constructor can't accept any parameters.
