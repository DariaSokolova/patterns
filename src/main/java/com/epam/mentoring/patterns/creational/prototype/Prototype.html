Prototype

Description

Specifies the kinds of objects to create using a prototypical instance, and create new objects by copying this prototype.
Should be used when a system should be independent of how its products are created, composed, and represented; to avoid building a class hierarchy of factories that parallels the class hierarchy of products; when instances of a class can have one of only a few different combinations of state.

Using in java API:

java.text.Format#clone()
java.util.Date#clone()

Pros:

  • Hides the concrete product classes from the client.
  • Specifying new objects by varying values and structure.
  • Reduces subclassing.
  • Let a client work with application-specific classes without modification.

Cons:

  • Classes that have circular references can be difficult to deep clone.
